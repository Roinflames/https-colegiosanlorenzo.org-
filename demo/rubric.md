# Rubric for HTML Design and Development Assignments

## Assignment: [Name of Assignment]

| Category          | 4 - Excellent                                                               | 3 - Good                                                                        | 2 - Satisfactory                                                               | 1 - Needs Improvement                                                          |
| :---------------- | :-------------------------------------------------------------------------- | :------------------------------------------------------------------------------ | :----------------------------------------------------------------------------- | :----------------------------------------------------------------------------- |
| **HTML Structure**| Uses semantic HTML5 tags appropriately. Clean, well-nested, and valid HTML. | Uses mostly semantic HTML5, minor issues with nesting or validation.            | Basic HTML structure, some non-semantic tags used, multiple validation errors. | Poor HTML structure, extensive use of `div` for layout, many validation errors. |
| **CSS Styling**   | Effective and creative use of CSS for layout and presentation. Responsive design implemented. External stylesheet used. | Good CSS use, some responsiveness, minor styling inconsistencies. External stylesheet used. | Basic styling, inconsistent layout, limited responsiveness. Internal/inline styles prevalent. | Minimal or no styling, poor layout, not responsive.                            |
| **Content**       | All required content is present, well-organized, clear, and relevant.       | Most required content present, generally well-organized and clear.              | Some required content missing or poorly organized, lacking clarity.            | Missing significant content, disorganized, unclear.                            |
| **Interactivity** | All interactive elements (forms, links, JS) function correctly and enhance user experience. | Most interactive elements function correctly, minor UX issues.                | Some interactive elements present but with functional or UX issues.            | Few or no interactive elements, or significant functional problems.             |
| **Best Practices**| Code is clean, well-commented (where necessary), and accessible. Follows coding conventions. | Code is generally clean, some comments, minor accessibility issues.           | Code is messy, few comments, significant accessibility concerns.               | Code is very difficult to read, no comments, major accessibility problems.     |

## Grading Scale (60% Exigency)

To achieve a passing grade (4.0 or higher), the student must demonstrate a minimum of 60% proficiency across all categories.

| Percentage Range | Grade |
| :--------------- | :---- |
| 90-100%          | 7.0   |
| 85-89%           | 6.0   |
| 75-84%           | 5.0   |
| 60-74%           | 4.0   |
| 50-59%           | 3.0   |
| 40-49%           | 2.0   |
| 0-39%            | 1.0   |
